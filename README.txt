============================================================
  ScoutMind - AI Meeting Planner for the Lebanese Scouts Association
============================================================

ScoutMind is a multi-agent AI system that generates professional,
pedagogically structured weekly meeting plans for Lebanese Scout units.
It uses LangGraph to orchestrate six specialised AI agents, a RAG
knowledge base of scouting activities and educational techniques,
and live contextual data (weather, Lebanese calendar).

------------------------------------------------------------
SYSTEM REQUIREMENTS
------------------------------------------------------------
  - Python 3.10 or higher
  - pip (Python package manager)
  - Internet connection (for Anthropic API and weather API)
  - ~500 MB disk space (sentence-transformer model cache)

------------------------------------------------------------
INSTALLATION
------------------------------------------------------------

1. Clone or extract the project to a folder on your machine.

2. Create and activate a virtual environment (recommended):

     python -m venv venv

     Windows:   venv\Scripts\activate
     Mac/Linux: source venv/bin/activate

3. Install all dependencies:

     pip install -r requirements.txt

   The first run will download the sentence-transformer model
   (~80 MB, cached automatically after first download).

4. Install the project as an editable package (required once):

     pip install -e .

   This lets Python resolve internal imports (agents, rag, tools, etc.)
   from any working directory without path hacks.

------------------------------------------------------------
CONFIGURATION
------------------------------------------------------------

1. Copy the example file and fill in your keys:

     cp .env.example agents/.env

   Then edit agents/.env and replace the placeholders:

     ANTHROPIC_API_KEY=[INSERT ANTHROPIC API KEY HERE]
     OPENWEATHER_API_KEY=[INSERT OPENWEATHERMAP API KEY HERE]

2. Get your API keys:
   - Anthropic API key: https://console.anthropic.com/
   - OpenWeatherMap key: https://openweathermap.org/api (free tier is sufficient)

------------------------------------------------------------
BUILDING THE KNOWLEDGE BASE
------------------------------------------------------------

The knowledge base must be built before running the application.

Step 1 - Build ChromaDB from the existing knowledge base:

     python rag/embeddings.py

Step 2 (optional) - Generate additional activities and techniques
using Claude (takes ~5-10 minutes, costs ~$0.50 in API credits):

     python rag/generate_kb.py --rebuild

   This expands the knowledge base from ~25 to ~175 activities
   and from 12 to ~24 educational techniques, significantly
   improving the quality of RAG retrieval.

------------------------------------------------------------
RUNNING THE APPLICATION
------------------------------------------------------------

     streamlit run ui/app.py

The app will open automatically in your browser at:
     http://localhost:8501

------------------------------------------------------------
AGENT PIPELINE OVERVIEW
------------------------------------------------------------

Each meeting plan is generated by six agents in sequence:

  1. Context Awareness Agent
     Checks today's weather in Beirut (OpenWeatherMap) and
     detects Lebanese public holidays and scout occasions.

  2. Educational Design Agent
     Designs the activity sequence structure using educational
     therapy frameworks (zone of proximal development, cognitive
     load theory, energy management).

  3. Scouting Context Agent
     Queries the RAG knowledge base (ChromaDB + sentence-transformers)
     to retrieve activities and techniques relevant to the theme
     and unit. Selects the best matches using Claude.

  4. Activity Generator Agent
     Writes full descriptions for each selected activity:
     objectives, step-by-step instructions, materials, leader tips,
     theme connections, and educational technique annotations.

  5. Validation Agent
     Validates the plan against structural rules (timing balance,
     energy bookends, cognitive load variety). Uses the Time Validator
     tool to auto-correct duration mismatches.

  6. Formatting Agent
     Assembles the final plan with a timestamped schedule, master
     materials list, and contextual advisories. Scores the plan
     using the Plan Evaluator (0-100 quality score).

------------------------------------------------------------
RUNNING TESTS / STANDALONE AGENTS
------------------------------------------------------------

Each agent and tool can be run independently for testing:

     python agents/orchestrator.py          # full pipeline test
     python agents/context_awareness.py     # weather + calendar test
     python agents/educational_design.py    # sequence design test
     python agents/validation.py            # validation rule test
     python tools/time_validator.py         # timing correction test
     python tools/plan_evaluator.py         # quality scoring test
     python rag/embeddings.py --force       # rebuild ChromaDB

------------------------------------------------------------
TROUBLESHOOTING
------------------------------------------------------------

"ChromaDB collection is empty"
  -> Run: python rag/embeddings.py

"ANTHROPIC_API_KEY not set"
  -> Check that agents/.env exists and contains the correct key.

"ChromaDB files locked"
  -> Stop Streamlit (Ctrl+C) before rebuilding: python rag/embeddings.py --force

"sentence_transformers not found"
  -> Run: pip install sentence-transformers

Slow first generation
  -> The sentence-transformer model loads on first use (~5 seconds).
     Subsequent calls reuse the loaded model (cached in memory).
